Elasticsearch simple introduction

简单地说， Elasticsearch 是一个使用 REST 接口的分布式的搜索引擎。

REST: 可以通过简单的 RESTful API、各种语言的客户端甚至命令行与之交互.

  1. Index：Elasticsearch的Index相当于数据库的Table
  2. Document：Document相当于数据库的一行记录
  3. Field：相当于数据库的Column的概念
  4. Mapping：相当于数据库的Schema的概念
  5. DSL(Domain Specific Languag,领域特定语言)：相当于数据库的SQL（给我们读取Elasticsearch数据的API）

分布式及高可用性的搜素引擎

  1. 每个索引（index）都使用可配置数量的分片进行完全分片
  2. 每个分片都可以有一个或多个副本
  3. 在任何副本分片上可执行读取/搜索操作

ElasticSearch _source字段设置false导致的查询不出数据内容问题

Posted on 2022-08-08 Edited on 2022-08-15 In ElasticSearch

在对ElasticSearch进行查询的时候，查询结果如下

可以看到查询结果中只有基本的三个字段和score评分，而查阅es的文档可以知道，

The _source field contains the original JSON document body that was passed at index time. The _source field itself is not indexed (and thus is not searchable), but it is stored so that it can be returned when executing fetch requests, like get or search.

意思是_source字段包含了原始的JSON文档内容，当上面的_source的enable属性设置为false时，会导致不会存储原始的json文档内容。而es是使用的倒排索引。

对于例子中两条数据，es存储如下

term docs
Jack 1
15 1
hiking 1
basketball 1,2
Tom 2
16 2
football 2
swiming 2

当_source的属性enabled为true的时候，会额外存储如下数据

doc term
1 Jack,15,[hiking,basketball]
2 Tom,16,[football,swiming,basketball]
所以，当enable设置为false的时候，查询仍然可用，只是不会显示完整的数据信息。
